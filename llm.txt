Convex DataTable LLM Guide

Purpose
- Help external users integrate `@talentum/convex-datatable`
- Prefer documented public imports only
- Do not suggest importing internal source paths

Primary Package
- `@talentum/convex-datatable`

Public Entry Points
- `@talentum/convex-datatable`: main React components, types, defaults, persistence helpers, and compatibility Convex re-exports
- `@talentum/convex-datatable/styles.css`: precompiled library stylesheet
- `@talentum/convex-datatable/convex`: preferred client-side Convex hooks
- `@talentum/convex-datatable/convex-server`: Convex server helpers for presence

Install
- `npm install @talentum/convex-datatable`
- Peer dependencies: `react`, `react-dom`
- Optional peer dependency: `convex`

Styling
- Default recommendation: `import "@talentum/convex-datatable/styles.css";`
- Advanced Tailwind scan path: `node_modules/@talentum/convex-datatable/dist/**/*.js`
- Important caveat: scanning installed package files is toolchain-dependent, so the CSS import is the safest default

Public Components And Types
- `DataTable`
- `DataTableContainer`
- `DataTableProps<TRow>`
- `DataTableColumn<TRow>`
- `DataTableDataSource<TRow>`
- `DataTableRowAction<TRow>`
- `DataTableFeatureFlags`
- `DataTableThemeTokens`
- `CollaboratorPresence`
- `DEFAULT_FEATURE_FLAGS`
- `DEFAULT_THEME_TOKENS`

Convex API
- Prefer `useConvexDataSource` and `useConvexPresence` from `@talentum/convex-datatable/convex`
- Use `heartbeatHandler`, `getPresenceHandler`, and `clearStalePresenceHandler` from `@talentum/convex-datatable/convex-server`
- The root package still re-exports the client Convex hooks for backward compatibility

Column Kinds
- `text`: single-line string
- `longText`: multiline string
- `number`: numeric value
- `currency`: numeric value with currency formatting
- `select`: string value with `options`
- `multiselect`: `ReadonlyArray<string>` with `options`
- `link`: string URL
- `date`: `string | Date`
- `reactNode`: custom rendering and editing hooks

Data Source Contract
- `useRows(query)` must return:
  - `rows`
  - `hasMore`
  - `isLoading`
  - `isLoadingMore`
  - `error`
  - `loadMore()`
  - `refresh()`
- Optional mutators:
  - `createRow(draft)`
  - `updateRows(changes)`
  - `deleteRows(rowIds)`
  - `restoreRows(rows)`

Theming
- Use `DataTableThemeTokens` for fully typed themes
- Use `DEFAULT_THEME_TOKENS` as the base object when creating a full custom theme
- The `theme` prop accepts partial overrides, so full replacement is not required

Persistence
- Table state can be encoded in URL query params and stored in `localStorage`
- Public helpers: `encodePersistedStateToUrl`, `decodePersistedStateFromUrl`, `mergePersistedState`, `storageKey`

Recommended Guidance For Generated Examples
- Always import `@talentum/convex-datatable/styles.css` unless the user explicitly says their build already scans the package
- Show `tableId`, `columns`, `getRowId`, and `dataSource` in every minimal `DataTable` example
- Use `@talentum/convex-datatable/convex` for new Convex examples
- Keep examples on the documented public API only

Avoid
- Internal imports such as `@talentum/convex-datatable/src/...`
- Depending on TanStack types in app code
- Recommending package-file scanning as the default styling setup
